# PRD — 열수송관 IoT 온도센서 모니터링 웹앱

| 항목 | 내용 |
|---|---|
| 문서명 | 열수송관 IoT 온도센서 이상감지 모니터링 웹앱 |
| 작성일 | 2026-06-08 |
| 버전 | v0.1 (초안) |
| 작성 배경 | 한국지역난방공사 열수송관 관리업무 — 센서 데이터 모니터링 도구화 |

---

## 1. 개요

한국지역난방공사 열수송관에 설치된 IoT 온도센서의 측정 데이터를 업로드하여, **기준 온도를 초과한 센서를 신속히 식별·경보**하는 단일 HTML 웹앱이다. 별도 서버 구축 없이 브라우저에서 동작하며, 현장 담당자가 CSV/엑셀 파일만 있으면 즉시 점검할 수 있도록 한다.

## 2. 목적 및 목표

- **1차 목적:** 이상 감지·경보. "어느 센서가 기준을 넘었는가"를 한눈에 파악.
- **부가 목적:** 센서별 온도 추이 확인, 위치 기반 직관적 상황 인지.
- **성공 기준:** 파일 업로드 후 별도 조작 없이 경보 대상 센서가 즉시 목록·지도·색상으로 드러난다.

## 3. 대상 사용자

- 열수송관 관리 담당자 (1차 사용자)
- 데이터 점검·보고가 필요한 운영 인력

## 4. 입력 데이터

### 4.1 공급 방식
- CSV 또는 엑셀(.xlsx) **파일 업로드** 방식.
- 실시간 API 연동은 본 버전 범위에서 제외(향후 확장).

### 4.2 데이터 구조 (한 행 = 한 측정 기록)

| 필드 | 설명 | 비고 |
|---|---|---|
| 센서 ID / 이름 | 센서 식별값 | 필수 |
| 측정 시각 | 날짜·시간 | 필수, 추이 그래프 X축 |
| 온도 값 | 측정 온도(°C) | 필수, 경보 판정 대상 |
| 위치 정보 | 위도·경도 좌표 | 필수, 지도 표시용 |

- 한 센서가 여러 시각의 기록(시계열)을 가질 수 있음.

## 5. 경보 로직

- **2단계 임계치:**
  - 경고(Warning): 40°C 이상 → 노란색
  - 위험(Danger): 50°C 이상 → 빨간색
- **판정 기준:** 각 센서의 **가장 최근 측정값**을 기준으로 현재 경보 상태를 결정한다.
- **임계치 조정:** 기본값 40 / 50°C, 화면에서 숫자 입력으로 수정 가능.

## 6. 기능 요구사항

### 6.1 파일 업로드
- CSV / .xlsx 업로드 지원.
- 일반적인 헤더명(한글·영문)을 자동 인식하여 컬럼 매핑.
- 잘못된 형식·누락 시 안내 메시지 표시.

### 6.2 지도 뷰
- 센서 위치를 위경도 좌표로 지도에 표시.
- 경보 상태에 따라 마커 색상 구분(정상/경고/위험, 위험 시 빨간색).
- 마커 클릭 시 해당 센서 정보 확인.
- 구현: Leaflet + OpenStreetMap (타일 로딩에 인터넷 연결 필요).

### 6.3 경보 목록
- 기준 온도(40°C)를 초과한 센서만 모아서 표시.
- 센서명, 최근 온도, 측정 시각, 경보 단계(경고/위험) 노출.
- 위험 단계가 상단에 오도록 정렬.

### 6.4 센서별 온도 추이 그래프
- 선택한 센서의 전체 측정 이력을 시계열 그래프로 표시.
- 40°C / 50°C 기준선 함께 표기.

## 7. 비기능 요구사항

- **형태:** 단일 HTML 파일(별도 설치·서버 불필요).
- **언어:** 한국어 UI.
- **톤:** 공사 환경에 맞는 차분하고 신뢰감 있는 톤.
- **네트워크:** 지도 타일·라이브러리 로딩 시 인터넷 필요. 그 외 데이터 처리는 브라우저 내(로컬).
- **데이터 처리:** 업로드된 데이터는 외부로 전송하지 않고 브라우저 내에서만 처리.

## 8. 가정 및 제약

1. 경보 상태는 센서의 **최근 1개 측정값** 기준으로 판정한다.
2. 추이 그래프는 해당 센서의 **전체 이력**을 보여준다.
3. 지도는 Leaflet + OpenStreetMap을 사용하며 타일 로딩에 인터넷이 필요하다.
4. 실제 CSV가 없어도 테스트할 수 있도록 **샘플 데이터**를 내장한다.
5. 위치 정보는 위경도 좌표 형식임을 전제로 한다.

## 9. 범위 외 (향후 확장 후보)

- 실시간 서버/API 연동 (자동 갱신)
- 전체 데이터 표(정렬·검색) 뷰
- 경보 발생 시 소리·알림·이메일 등 능동적 통지
- 변화율(급변) 기반 이상 감지, 센서 간 온도 차 기반 누수 추정
- 경보 이력 저장 및 보고서 출력

## 10. 미해결/추후 결정 사항

- 실제 CSV 헤더 명칭 확정 (자동 매핑 정확도와 직결).
- 정상/경고/위험 외 추가 상태(센서 미응답, 데이터 결측 등) 처리 여부.
- 측정 시각의 형식(날짜·시간 표기 규칙) 표준화.
